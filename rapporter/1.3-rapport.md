### 2. Valgt problemstilling

**Forskningsspørsmål:**
Gjør det en forskjell i bruk av Claude Code om man har erfaring med verktøyet fra før? Og: gir samme enkle prompt svært ulike resultater på tvers av AI-modeller?

**Hypotese:**
Erfarne brukere utnytter Claude Code mer effektivt – bedre prompts, kortere iterasjonsløkker, og resultater som er lettere å stole på. For modellsammenligningen: alle produserer noe fungerende, men kvalitet og stil vil variere.

---

### 3. Eksperimentoppsett

#### Hva ble testet

To eksperimenter ble gjennomført:

**Eksperiment 1:** Alle fire gruppemedlemmer løste samme oppgave (lage et Snake-spill) fullt agentic – kun instrukser, ingen manuell koding. Verktøy og erfaringsnivå varierte.

**Eksperiment 2:** 19 ulike AI-modeller fikk nøyaktig samme prompt og produserte hvert sitt Snake-spill. Kjørt parallelt via OpenCode med GitHub Copilot provider.

#### Betingelser

**Eksperiment 1:**

| Betingelse | Verktøy | Erfaring | Deltaker |
|---|---|---|---|
| A – Erfaren | Claude Code CLI | Daglig bruk | Felix |
| B – Erfaren | Claude Code CLI | Daglig bruk | Rafael |
| C – Lite erfaren | Claude Code CLI | Brukt litt | Rune |
| D – Lite erfaren (kontroll) | JetBrains AI + Claude | Brukt litt | Vegard |

**Eksperiment 2:** Prompt: *"Create a single self-contained HTML file with a fully playable Snake game."* – 19 modeller: Claude (haiku/opus/sonnet, ulike versjoner), GPT-5.x, Gemini 2.5/3.x, Grok.

#### Målemetoder

- Om spillet kom til å kjøre etter første prompt
- Antall iterasjoner/prompts
- Promptestrategi (bred vs. inkrementell)
- Visuell og funksjonell kvalitet på resultatet
- Tokenbruk og tidsbruk (eks. 2)

---

### 4. Resultater

**Eksperiment 1:**

- Alle fire fikk et kjørende Snake-spill etter første prompt
- Tre deltakere (Felix, Rafael, Rune) ga én bred instruksjon og fikk nesten identiske spill
- Vegard styrte agenten trinn for trinn (git repo → filstruktur → funksjonalitet) og fikk et annerledes resultat
- Opprinnelig plan om å teste ulike nivåer av agentic ble forkastet – alle valgte fullt agentic av seg selv

**Eksperiment 2:**

| Kategori | Modell | Begrunnelse |
|---|---|---|
| Mest polert | gpt-5.2 | 841 linjer, ARIA, auto-pause, DPR-aware canvas |
| Beste arkitektur | claude-opus-4.7 | State machine, rAF, pause, keyboard hints |
| Mest kompakt | claude-opus-4.6 | 124 linjer, fullt spillbar |
| Raskest | gemini-3-flash-preview | 27 sekunder |
| Mest unik | gpt-5.2-codex | Wrap-around vegger, "Neon Serpent"-branding |

- 16/19 produserte spillbare spill
- 2 feilet helt (gpt-4.1, gpt-4o – ingen output)
- 1 ødelagt (grok-code-fast-1 – self-collision bug ved oppstart)
- Modellene klustrer i tydelige stilgrupper (f.eks. claude-sonnet-4 og sonnet-4.5 er nesten identiske)

---

### 5. Diskusjon

#### Hva funket

- Fullt agentic fungerte overraskende bra for alle deltakere, uavhengig av erfaring
- OpenCode som benchmark-verktøy fungerte godt for parallell kjøring av 19 modeller
- Snake som oppgave var god nok til å skille modeller fra hverandre (eks. 2), selv om den var for enkel til å skille erfaringsnivåer (eks. 1)

#### Hva funket ikke

- Eksperiment 1 klarte ikke å teste den opprinnelige hypotesen – oppgaven var for enkel til at erfaring utgjorde en målbar forskjell
- Erfaringsforskjellen viste seg ikke i resultatet, men i tilnærmingen: Vegard valgte inkrementell styring naturlig, uten at det var et bevisst metodisk valg

#### Begrensninger

- For få deltakere til å trekke statistiske slutninger
- Snake er en veldefinert, enkel oppgave – resultater kan ikke generaliseres til komplekse eller uklare oppgaver
- Vegards avvikende tilnærming var ikke planlagt, noe som gjør det vanskelig å isolere årsaken til forskjellen

---

### 6. Konklusjon

Promptestrategi ser ut til å ha større innvirkning på resultatet enn både verktøyvalg og tidligere erfaring med Claude Code – i hvert fall for enkle, veldefinerte oppgaver. Når oppgaven er tilstrekkelig enkel, leverer alle modeller og alle brukere noe fungerende på første forsøk, men inkrementell styring av agenten gir et annerledes og muligens bedre strukturert resultat enn én bred bestilling. Modellsammenligningen (eks. 2) viser at kvalitet, arkitektur og særpreg varierer betydelig på tvers av modeller – selv med identisk prompt.
